# Система тарифов подписки School Club

## Обзор

Приложение School Club теперь использует **трехуровневую систему тарифов** вместо простой системы "бесплатно/премиум".

## Тарифные планы

### 1. BASE (Базовый) 
**Бесплатный тариф**

- ✅ Пробные уроки
- ✅ Просмотр рекламы
- ✅ Базовые функции приложения
- ❌ Доступ к полным урокам
- ❌ Чаты группы
- ❌ Успеваемость

**Назначение:** Для всех новых пользователей при регистрации. Позволяет ознакомиться с приложением.

---

### 2. STUDENT (Студент)
**Для студентов School Club**

- ✅ Всё из BASE
- ✅ Чат "Моя группа"
- ✅ Доступ к успеваемости
- ✅ Расписание занятий
- ❌ Полный доступ к урокам
- ❌ AI ассистент
- ❌ Форум

**Назначение:** Только для зарегистрированных студентов School Club. Назначается администратором.

**Как получить:** Записаться в School Club через https://schoolclub.kg/enroll

---

### 3. PRO (Профессиональный)
**Платная подписка - полный доступ**

- ✅ Всё из STUDENT (если студент)
- ✅ Все уроки и тесты
- ✅ Форум
- ✅ AI ассистент
- ✅ Личные чаты
- ✅ Без рекламы
- ✅ Приоритетная поддержка

**Стоимость:**
- 500 сом / месяц
- 4000 сом / 9 месяцев (ОРТ) - выгодно!

**Назначение:** Для пользователей, желающих получить максимальный доступ ко всем функциям.

**Как получить:** Купить подписку в приложении через раздел "Профиль" → "Улучшить подписку"

## Изменения в коде

### 1. Модель User (User.swift)

Добавлено:
```swift
enum SubscriptionTier: String, Codable {
    case base = "base"
    case student = "student"
    case pro = "pro"
}

struct User {
    var subscriptionTier: SubscriptionTier  // Новое поле
    
    // Вспомогательные свойства
    var hasStudentAccess: Bool  // true для student и pro
    var hasProAccess: Bool      // true только для pro
}
```

## Логика доступа к функциям

### Проверка доступа в коде:

```swift
// Проверка доступа студента
if user.hasStudentAccess {
    // Показать чат группы, успеваемость
}

// Проверка PRO доступа
if user.hasProAccess {
    // Показать уроки, AI, форум
}

// Проверка по конкретному тарифу
switch user.subscriptionTier {
case .base:
    // Только пробные материалы
case .student:
    // Чаты + успеваемость
case .pro:
    // Всё
}
```

## Firebase структура

В Firestore пользователь теперь имеет поле:
```json
{
  "uid": "...",
  "subscriptionTier": "base" | "student" | "pro",
  "isPremium": true/false,  // сохранено для обратной совместимости
  "premiumUntil": "2026-12-31T00:00:00Z"  // только для PRO
}
```

---

## Миграция существующих пользователей

При первой загрузке приложения с новой версией:
- Все существующие пользователи получат `subscriptionTier: .base`
- Пользователи с `isPremium: true` должны быть обновлены до `.pro`
- Студенты School Club должны быть обновлены администратором до `.student`
